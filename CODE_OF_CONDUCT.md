# Code of Conduct

Noema Atlas should be a practical, respectful place to collaborate. This code
applies to repository issues, pull requests, discussions, reviews, chats, events,
and other project spaces.

## Expected behavior

- Be direct, respectful, and constructive.
- Assume technical disagreement is normal, but keep criticism focused on the
  work and evidence.
- Make space for new contributors by answering questions clearly and linking to
  relevant docs.
- Credit prior work and disclose conflicts of interest when they matter.
- Respect maintainer decisions about scope, security, release timing, and
  project direction.

## Unacceptable behavior

- Harassment, threats, stalking, or sustained personal attacks.
- Discriminatory language or conduct.
- Sexualized attention or imagery in project spaces.
- Publishing private information without permission.
- Deliberately derailing issues, reviews, or security reports.
- Retaliation against people who report conduct or security concerns in good
  faith.

## Enforcement

Maintainers may remove comments, close threads, reject contributions, block
accounts, or take other proportionate action to protect the project.

Report conduct concerns privately to clientcare@noemaai.com with
`Noema Atlas conduct` in the subject. If the concern involves a security issue,
use the reporting path in [SECURITY.md](SECURITY.md). Reports will be handled
with care and shared only with people needed to resolve the situation.
